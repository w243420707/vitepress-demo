# VitePress Emoji 智能适配 Skill

## 功能说明

此 Skill 用于自动为 VitePress 导航栏标题添加合适的 emoji 图标，基于内容语义智能识别匹配。

## ⚠️ 重要规范

**Emoji 使用规则**：
- ✅ **主标题（分组标题）**：使用 emoji 图标
- ❌ **子项（导航项）**：使用序号格式，不使用 emoji
- ❌ **文件内标题**：使用序号格式，不使用 emoji

**序号格式规范**：
- 序号在每个分组内重新开始编号（每个分组从 01 开始）
- 格式：`XX- 标题`（两位数字 + 连字符 + 标题）
- 如果某个项目有子页面，才使用 `XX.X- 标题` 格式

**文件内标题序号规范**：
- 二级标题：`XX.X- 标题`（如：`## 01.1- 项目简介`）
- 三级标题：`XX.XX- 标题`（如：`### 01.11- 详细说明`）
- 四级标题：`XX.XXX- 标题`（如：`#### 01.111- 具体内容`）

**侧边栏示例**：
```javascript
sidebar: [
  {
    text: '📚 Git 教程',  // ✅ 主标题使用 emoji
    items: [
      { text: '01- Git 基础', link: '/git' },           // ❌ 子项使用序号（分组内从 01 开始）
      { text: '02- 网络环境配置', link: '/network' }    // ❌ 子项使用序号
    ]
  },
  {
    text: '🚀 GitHub 资源',  // ✅ 主标题使用 emoji
    items: [
      { text: '01- GitHub 使用指南', link: '/guide' },  // ❌ 子项使用序号（新分组从 01 开始）
      { text: '02- 全部资源概述', link: '/overview' }   // ❌ 子项使用序号
    ]
  }
]
```

**文件内标题示例**：
```markdown
# Git 基础

## 01.1- 项目简介

这是项目简介的内容...

## 01.2- 功能特性

### 01.21- 核心功能

核心功能的详细说明...

### 01.22- 扩展功能

扩展功能的详细说明...

## 01.3- 技术栈

技术栈的说明...
```

**子页面示例**：
```javascript
sidebar: [
  {
    text: '📚 Git 教程',
    items: [
      { text: '01- Git 基础', link: '/git' },
      // 如果 Git 基础有子页面
      { text: '01.1- Git 安装', link: '/git/install' },
      { text: '01.2- Git 配置', link: '/git/config' },
      { text: '02- 网络环境配置', link: '/network' }
    ]
  },
  {
    text: '🚀 GitHub 资源',
    items: [
      { text: '01- GitHub 使用指南', link: '/guide' },
      { text: '02- 全部资源概述', link: '/overview' }
    ]
  }
]
```

## 自动识别规则

AI 将根据以下维度自动选择 emoji：

1. **关键词匹配**：分析标题中的关键词
2. **语义理解**：理解标题的含义和上下文
3. **类别判断**：判断内容所属类别
4. **一致性原则**：保持同类内容使用相似的 emoji

## 自动适配工作流程

当用户要求添加导航项时，AI 应：

### 1. 分析标题内容

- 提取关键词
- 理解语义含义
- 判断内容类别

### 2. 匹配 Emoji

- 查找映射表中的关键词
- 选择最匹配的 emoji
- 确保语义一致性

### 3. 应用配置

- 自动在标题前添加 emoji
- 保持格式统一：`emoji + 空格 + 标题`

### 4. 验证结果

- 检查 emoji 是否与内容相关
- 确保同类内容使用相似 emoji
- 保持整体风格一致

## 智能识别示例

### 示例 1：添加新教程

```
用户：添加"Python 入门教程"
AI 自动识别：
- 关键词：教程、入门
- 类别：教程类
- 匹配 Emoji：📚
- 结果：📚 Python 入门教程
```

### 示例 2：添加新资源

```
用户：添加"开源项目推荐"
AI 自动识别：
- 关键词：开源、项目
- 类别：资源类
- 匹配 Emoji：🚀（分组）或 📦（项目）
- 结果：🚀 开源项目推荐
```

### 示例 3：添加网络相关

```
用户：添加"代理配置"
AI 自动识别：
- 关键词：代理、配置
- 类别：网络/配置
- 匹配 Emoji：🌐
- 结果：🌐 代理配置
```

## 特殊情况处理

### 1. 多关键词匹配

- 优先匹配更具体的关键词
- 例如："网络配置教程" → 🌐（网络优先）

### 2. 无明确关键词

- 根据上下文和语义判断
- 默认使用通用 emoji：📄（文档）

### 3. 自定义 Emoji

- 用户明确指定时，使用用户指定的 emoji
- 优先级：用户指定 > AI 自动识别

## 一致性检查

AI 在添加导航项后应检查：

- ✅ 同一分组下的项目使用相似的 emoji 风格
- ✅ 避免同一页面出现重复 emoji
- ✅ 确保 emoji 与内容语义相关
- ✅ 保持整体视觉和谐

## 使用方法

当需要为导航栏添加 emoji 时，AI 会自动：

1. 分析标题内容和关键词
2. 查询此映射表
3. 选择最合适的 emoji
4. 应用到配置文件中

## 注意事项

- 保持 emoji 的语义一致性
- 避免过度使用 emoji
- 确保可读性和美观性
- 定期更新映射表以适应新内容
