# VitePress 构建前整理清单

## ⚠️ 强制要求

**🚨 重要：每次构建前必须强制执行以下整理步骤，不得跳过！**

**AI 在执行构建命令前，必须先完成所有整理检查项，否则不允许执行构建。**

## 📋 整理清单（必须逐项完成）

### 1. 导航栏 Emoji 检查

- ✅ 检查主标题（分组标题）是否已添加 emoji
- ✅ 检查子项是否使用序号格式（每个分组从 01 开始）
- ✅ 检查子页面是否使用正确的序号格式（01.1-、01.2-...）
- ✅ 检查文件内标题是否使用序号格式（二级：01.1-，三级：01.11-）
- ✅ 确认 emoji 与内容语义匹配
- ✅ 调用 `vitepress-emoji-adapter` Skill 自动适配

### 2. 侧边栏配置检查

- ✅ 确认所有分组都有总述页面（link 属性）
- ✅ 检查所有链接是否有效
- ✅ 验证侧边栏结构与导航栏一致

### 3. 文档完整性检查

- ✅ 所有 markdown 文件都有标题和描述
- ✅ 检查文档内部链接是否正确
- ✅ 确认图片资源是否存在

### 4. 多语言适配检查

- ✅ 检查所有页面是否有对应的英文版本
- ✅ 检查英文导航栏配置是否完整
- ✅ 检查英文侧边栏配置是否完整
- ✅ 检查英文页面内容是否正确翻译
- ✅ 调用 `vitepress-i18n` Skill 自动适配

### 5. 配置文件检查

- ✅ `.vitepress/config.mjs` 语法正确
- ✅ 所有路径引用正确
- ✅ 多语言配置完整（如有）

### 5. 代码规范检查

- ✅ JavaScript 代码符合规范
- ✅ Markdown 格式正确
- ✅ 无语法错误

## 🔧 自动修复机制

**当检查不通过时，AI 必须自动执行修复，无需用户确认。**

### 修复规则

#### 1. 导航栏 Emoji 检查失败

- ❌ 问题：主标题缺少 emoji 或子项序号格式错误
- 🔧 自动修复：
  - 调用 `vitepress-emoji-adapter` Skill
  - 为分组标题添加匹配的 emoji
  - 为子项添加正确的序号格式（每个分组从 01 开始）
  - 为子页面添加正确的序号格式（如 01.1-、01.2-）
  - 为文件内标题添加正确的序号格式（二级：01.1-，三级：01.11-）
  - 更新 `.vitepress/config.mjs` 和相关 markdown 文件

#### 2. 侧边栏配置检查失败

- ❌ 问题：侧边栏分组缺少总述页面
- 🔧 自动修复：
  - 创建对应的总述 markdown 文件
  - 添加分组描述和子项目链接
  - 在侧边栏配置中添加 `link` 属性
  - 更新 `.vitepress/config.mjs`

#### 3. 文档完整性检查失败

- ❌ 问题：markdown 文件缺少标题或描述
- 🔧 自动修复：
  - 为文档添加一级标题
  - 添加 frontmatter 描述
  - 补充缺失的内容结构

#### 4. 多语言适配检查失败

- ❌ 问题：缺少英文版本或英文配置不完整
- 🔧 自动修复：
  - 调用 `vitepress-i18n` Skill
  - 为所有中文页面创建对应的英文版本
  - **翻译所有内容，无一例外**（包括UI界面和广告弹窗）：
    - 文档内容（标题、段落、列表、表格、引用等）
    - 代码注释和文档字符串
    - 配置文件中的所有文字（导航栏、侧边栏、站点标题等）
    - UI界面所有元素（搜索框、页脚、分页导航、编辑链接等）
    - 广告弹窗内容（标题、描述、按钮、优惠券信息）
    - Frontmatter 元数据
    - 图片 alt 文本
    - 自定义组件中的文字
    - JavaScript 提示和弹窗文字
  - 保持原始格式和结构不变
  - 更新英文导航栏配置
  - 更新英文侧边栏配置
  - 更新 `.vitepress/config.mjs` 的 locales 配置
  - 更新广告弹窗代码中的文字内容

#### 5. 配置文件检查失败

- ❌ 问题：配置文件语法错误
- 🔧 自动修复：
  - 修复 JavaScript 语法错误
  - 修正路径引用
  - 补全缺失的配置项

#### 6. 代码规范检查失败

- ❌ 问题：代码不符合规范
- 🔧 自动修复：
  - 格式化代码
  - 修正命名规范
  - 修复语法错误

## 🔄 执行流程

```
用户请求构建
    ↓
AI 读取整理清单
    ↓
逐项执行检查
    ├─ 检查 1: Emoji ✅
    ├─ 检查 2: 侧边栏 ✅
    ├─ 检查 3: 文档 ✅
    ├─ 检查 4: 多语言适配 ✅
    ├─ 检查 5: 配置 ✅
    └─ 检查 6: 代码 ✅
    ↓
所有检查通过？
    ├─ 是 → 执行构建 ✅
    └─ 否 → 自动修复问题 → 重新检查 🔄
```

## 🔧 自动修复流程

```
检查项失败
    ↓
识别问题类型
    ↓
执行自动修复
    ├─ 读取相关文件
    ├─ 分析问题原因
    ├─ 应用修复方案
    └─ 保存修改
    ↓
重新执行检查
    ↓
检查通过？
    ├─ 是 → 继续下一项检查 ✅
    └─ 否 → 再次修复 🔄
```

## 🎯 修复优先级

### 高优先级（立即修复）

- 配置文件语法错误
- 缺少必要的文件
- 路径引用错误
- 多语言配置缺失

### 中优先级（自动修复）

- 缺少 emoji
- 缺少总述页面
- 文档结构不完整
- 英文内容未翻译

### 低优先级（提示用户）

- 内容质量优化
- 性能改进建议

## 📋 修复日志

AI 在修复过程中应记录：

```
🔍 检查发现：导航栏缺少 emoji
🔧 执行修复：调用 vitepress-emoji-adapter Skill
✅ 修复完成：已添加 emoji 图标
🔄 重新检查：检查通过
```

## ⚠️ 注意事项

- **🚨 禁止跳过整理步骤**：直接构建可能导致错误和资源缺失
- **📋 必须逐项检查**：按照清单逐项确认，不得遗漏任何检查项
- **✅ 全部通过才能构建**：只有所有检查项都通过后，才能执行构建命令
- **🔧 自动修复问题**：检查失败时，AI 必须自动修复，无需用户确认
- **🔄 修复后重新检查**：修复完成后，必须重新执行检查，确保问题已解决
- **🧪 测试验证**：在开发模式下充分测试后再构建
- **💾 保存更改**：确保所有更改已保存到文件
