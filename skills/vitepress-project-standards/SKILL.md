# VitePress 项目开发规范

## 核心规范

### 1. 导航栏 Emoji 智能适配

**⚠️ 重要规范**：
- ✅ **主标题（分组标题）**：使用 emoji 图标
- ❌ **子项（导航项）**：使用序号格式，不使用 emoji
- ❌ **文件内标题**：使用序号格式，不使用 emoji

**详细规则**：调用 `vitepress-emoji-adapter` Skill

**序号格式规范**：
- 序号在每个分组内重新开始编号（每个分组从 01 开始）
- 格式：`XX- 标题`（两位数字 + 连字符 + 标题）
- 如果某个项目有子页面，才使用 `XX.X- 标题` 格式

**文件内标题序号规范**：
- 二级标题：`XX.X- 标题`（如：`## 01.1- 项目简介`）
- 三级标题：`XX.XX- 标题`（如：`### 01.11- 详细说明`）
- 四级标题：`XX.XXX- 标题`（如：`#### 01.111- 具体内容`）

**侧边栏示例**：
```javascript
sidebar: [
  {
    text: '📚 Git 教程',  // ✅ 主标题使用 emoji
    items: [
      { text: '01- Git 基础', link: '/git' },           // ❌ 子项使用序号（分组内从 01 开始）
      { text: '02- 网络环境配置', link: '/network' }    // ❌ 子项使用序号
    ]
  },
  {
    text: '🚀 GitHub 资源',  // ✅ 主标题使用 emoji
    items: [
      { text: '01- GitHub 使用指南', link: '/guide' },  // ❌ 子项使用序号（新分组从 01 开始）
      { text: '02- 全部资源概述', link: '/overview' }   // ❌ 子项使用序号
    ]
  }
]
```

**文件内标题示例**：
```markdown
# Git 基础

## 01.1- 项目简介

这是项目简介的内容...

## 01.2- 功能特性

### 01.21- 核心功能

核心功能的详细说明...

### 01.22- 扩展功能

扩展功能的详细说明...

## 01.3- 技术栈

技术栈的说明...
```

**子页面示例**：
```javascript
sidebar: [
  {
    text: '📚 Git 教程',
    items: [
      { text: '01- Git 基础', link: '/git' },
      // 如果 Git 基础有子页面
      { text: '01.1- Git 安装', link: '/git/install' },
      { text: '01.2- Git 配置', link: '/git/config' },
      { text: '02- 网络环境配置', link: '/network' }
    ]
  },
  {
    text: '🚀 GitHub 资源',
    items: [
      { text: '01- GitHub 使用指南', link: '/guide' },
      { text: '02- 全部资源概述', link: '/overview' }
    ]
  }
]
```

**核心原则**：
- 🤖 AI 自动识别关键词和语义
- 📋 主标题使用 emoji，子项和文件内标题使用序号
- ✨ 保持一致性和语义相关性

**Emoji 快速参考（主标题）**：
- 📚 教程、学习类
- 🚀 资源、汇总类
- ⚙️ 配置、设置类
- 📖 指南、文档类
- 🛠️ 工具、插件类
- 💼 项目、案例类

### 2. Markdown 文档规范

#### 文件命名

- 使用小写字母和连字符：`github-guide.md`
- 中文文件名保持原样：`全部资源概述.md`
- 避免使用空格和特殊字符

#### 文档结构

每个文档应包含：
1. **标题**：使用 `#` 一级标题
2. **简介**：简要说明文档内容
3. **正文**：使用合适的标题层级
4. **相关链接**：文档末尾添加相关资源

#### Frontmatter

```yaml
---
title: 文档标题
description: 文档描述
---
```

### 3. 代码规范

#### JavaScript/TypeScript

- 使用 ES6+ 语法
- 使用 `const` 和 `let`，避免 `var`
- 函数和变量使用驼峰命名
- 常量使用大写字母和下划线

#### Markdown

- 使用 VitePress 扩展语法
- 使用自定义容器：`::: tip`、`::: warning`、`::: danger`
- 代码块指定语言类型
- 链接使用相对路径

### 4. Git 提交规范

#### 提交信息格式

```
<type>: <subject>

<body>
```

#### Type 类型

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `deploy`: 构建和部署

#### 示例

```
feat: 添加 GitHub 使用指南文档

- 创建 github-guide.md 文件
- 更新侧边栏配置
- 添加 emoji 图标
```

## 最佳实践

1. **保持一致性** - 遵循统一的规范和风格
2. **语义化设计** - 选择与内容相关的元素
3. **简洁明了** - 标题和描述应简洁易懂
4. **用户体验** - 确保导航结构清晰
5. **文档完善** - 每个页面都应有清晰的标题和描述
