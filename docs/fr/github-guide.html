Guide d'utilisation GitHub ​

01.1- Introduction ​

GitHub est la plus grande plateforme d'hébergement de code au monde, fournissant aux développeurs une gestion de versions puissante, un développement collaboratif et une gestion de projets.

01.2- Concepts de base ​

01.21- Dépôt ​

Un dépôt est l'espace de stockage d'un projet, contenant tous les fichiers, dossiers et l'historique des modifications de chaque fichier.

01.22- Branche ​

Une branche permet de développer de nouvelles fonctionnalités ou de corriger des bugs sans affecter la branche principale.

01.23- Commit ​

Un commit est un enregistrement des modifications d'un fichier, chaque commit se voit attribuer un identifiant unique.

01.24- Pull Request ​

Une pull request est une demande de fusion des modifications dans la branche principale, facilitant la révision du code et la discussion.

01.3- Démarrage rapide ​

01.31- Création d'un compte ​

Visitez le site officiel de GitHub pour vous inscrire.

01.32- Création d'un dépôt ​

  1. Cliquez sur le bouton "+" en haut
  2. Sélectionnez "New repository"
  3. Entrez le nom et la description du dépôt
  4. Choisissez public ou privé
  5. Cliquez sur "Create repository"

01.33- Clonage du dépôt ​

bash
git clone https://github.com/username/repository.git

01.34- Création d'une branche ​

bash
git checkout -b feature-branch

01.35- Commit des modifications ​

bash
git add .
git commit -m "Ajouter une nouvelle fonctionnalité"
git push origin feature-branch

01.36- Création d'une pull request ​

Créez une pull request sur la page GitHub et attendez la révision et la fusion.

01.4- Opérations courantes ​

01.41- Fork ​

Utilisez le fork pour copier le projet de quelqu'un d'autre dans votre compte et le modifier.

01.42- Étoile ​

L'étoile indique que vous aimez le projet, et vous pouvez le retrouver facilement plus tard.

01.43- Watch ​

Utilisez le watch pour recevoir les notifications de mise à jour du projet.

01.44- Issue ​

Les issues sont utilisées pour signaler des bugs, demander des fonctionnalités ou discuter d'idées.

01.5- Meilleures pratiques ​

  1. Messages de commit clairs - communiquez aux autres l'objectif de chaque commit
  2. Utilisation des branches - ne développez pas directement sur la branche principale
  3. Synchronisation régulière - synchronisez le dépôt local avec le dépôt distant
  4. Révision du code - effectuez des révisions de code via les pull requests
  5. Documentation du projet - ajoutez un README et la documentation nécessaire au projet
