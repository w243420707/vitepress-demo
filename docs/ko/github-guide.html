GitHub 사용 가이드 ​

01.1- 소개 ​

GitHub은 세계 최대의 코드 호스팅 플랫폼으로, 개발자에게 강력한 버전 관리, 협업 개발, 프로젝트 관리 기능을 제공합니다.

01.2- 기본 개념 ​

01.21- 저장소 ​

저장소는 프로젝트의 저장 공간으로, 모든 파일, 폴더 및 각 파일의 수정 이력을 포함합니다.

01.22- 브랜치 ​

브랜치를 사용하면 메인 브랜치에 영향을 주지 않고, 새 기능 개발이나 문제 수정이 가능합니다.

01.23- 커밋 ​

커밋은 파일에 대한 변경의 기록으로, 각 커밋에는 고유한 ID가 생성됩니다.

01.24- 풀 리퀘스트 ​

풀 리퀘스트는 변경사항을 메인 브랜치에 병합하는 요청으로, 코드 리뷰와 토론을 용이하게 합니다.

01.3- 퀵스타트 ​

01.31- 계정 생성 ​

GitHub 공식 사이트에 접속하여 계정을 등록하세요.

01.32- 저장소 생성 ​

  1. 우측 상단의 "+" 버튼 클릭
  2. "New repository" 선택
  3. 저장소 이름과 설명 입력
  4. 공개 또는 비공개 선택
  5. "Create repository" 클릭

01.33- 저장소 클론 ​

bash
git clone https://github.com/username/repository.git

01.34- 브랜치 생성 ​

bash
git checkout -b feature-branch

01.35- 변경사항 커밋 ​

bash
git add .
git commit -m "새 기능 추가"
git push origin feature-branch

01.36- 풀 리퀘스트 생성 ​

GitHub 웹페이지에서 풀 리퀘스트를 생성하고, 리뷰와 병합을 기다립니다.

01.4- 자주 사용하는 작업 ​

01.41- 포크 ​

포크를 사용하면 다른 사람의 프로젝트를 자신의 계정에 복사하여 수정할 수 있습니다.

01.42- 스타 ​

스타는 프로젝트가 마음에 든다는 것을 표시하며, 나중에 쉽게 찾을 수 있습니다.

01.43- 워치 ​

워치를 사용하면 프로젝트의 업데이트 알림을 받을 수 있습니다.

01.44- 이슈 ​

이슈는 문제 보고, 기능 요청 또는 아이디어 토론에 사용됩니다.

01.5- 모범 사례 ​

  1. 명확한 커밋 메시지 - 각 커밋의 목적을 다른 사람에게 전달
  2. 브랜치 사용 - 메인 브랜치에서 직접 개발하지 않기
  3. 정기적 동기화 - 로컬 저장소를 원격 저장소와 동기화
  4. 코드 리뷰 - 풀 리퀘스트를 통해 코드 리뷰 실시
  5. 문서화 - 프로젝트에 README와 필요한 문서 추가
